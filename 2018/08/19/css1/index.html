css中的居中和分栏问题总结

水平垂直居中

行内元素

  1. 设置text-align:center->水平居中，设置height=line-height->垂直居中。

  2. Flex布局

    /* 父元素 */
display: flex;
justify-content: center;

  3. table-cell

    display: table-cell;
vertical-align: middle;
text-align: center;

块级元素知道宽高实现水平垂直居中

  1. position:absolute + margin:auto
    将元素设置成position:absolute，并且将left、top、bottom、right设置为0，margin设置为auto。此时元素就会自动侵占父元素剩余空间，并且上/下，左/右占比相当，实现垂直水平居中。

    .test {
 width: 200px;
 height: 200px;
 background: green;
 position:absolute;
 left:0;
 top: 0;
 bottom: 0;
 right: 0;
 margin: auto;              
}

  2. position:absolute + 负margin
    将元素设置成position:absolute，并且将left和top设置成50%。此时元素的左上角为页面正中心，也就是说元素在页面正中心的右下方，偏移距离就是元素本身的宽/高。然后通过设置负的margin(绝对值为元素的宽/高的一半)，把元素“拉”回来，从而达到居中效果。

    .test {
 width:200px;
 height: 200px;
 background:green;
 position: absolute;
 left:50%;
 top:50%;
 margin-left:-100px;
 margin-top:-100px;               
}

  3. transform
    与absolute + 负margin相同，但是由于margin在不定义height/width的情况下会失效，所以无法适用于不知道宽高的场景。兼容性不好(IE8+)

    .test {
 width: 200px;
 height: 200px;
 background: green;
 position:absolute;
 left:50%;    /* 定位父级的50% */
 top:50%;
 transform: translate(-50%,-50%); /*自己的50% */
}

块级元素不知道宽高实现垂直水平居中(同样适用于知道宽高的场景)

  1. flex

    .test {
 height:600px;
 display:flex;
 justify-content:center;
 align-items:center;
 /* 只要三句话就可以实现不定宽高水平垂直居中 */
}

  2. table-cell
    兼容性好，但是性能差，浏览器绘制时间长，不利于SEO

    /*
table-cell实现居中
将父盒子设置为table-cell元素，设置
text-align:center,vertical-align: middle;
子盒子设置为inline-block元素
*/
.tableCell{
 display: table;
}
.tableCell .container{
 display: table-cell;
 text-align: center;
 vertical-align: middle;
}
.tableCell .innerBox{
 display: inline-block;
}
